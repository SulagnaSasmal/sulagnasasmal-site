About

Documentation that thinks like engineering.

I'm a Senior Technical Writer and Documentation Architect based in Pune, India, with 19+ years of experience across FinTech — payment processing, financial crime prevention, asset management, enterprise messaging, and SaaS platforms. My career spans Fundtech (payment hubs), SunGard (asset management), Avaya (enterprise communications), and NICE Actimize (financial crime prevention) — where I currently lead documentation for the Fraud and Compliance suite across on-prem and cloud-native deployments.

My work goes beyond writing. I architect documentation ecosystems, embed docs into CI/CD pipelines, influence portal roadmaps (search, filtering, content discoverability), and mentor teams of writers to deliver documentation that ships with — not after — the product.

I've also led the evaluation of 7 AI and productivity tools — building a governance framework for prompt standards, review checkpoints, and compliance controls — resulting in 4 tools formally adopted across a team of 20 writers.

I believe documentation is a product capability, not an afterthought. When it's done right, it reduces support load, accelerates developer adoption, and builds trust with enterprise clients.

RAG / AI
DocQuery — RAG Documentation Chatbot
Ask natural-language questions against documentation — with source citations, confidence scoring, and content gap detection
A complete RAG pipeline: crawl documentation sites, split content using section-aware chunking (respects heading hierarchy, never splits code blocks), embed with OpenAI text-embedding-3-small, and store in ChromaDB. Users ask questions and get answers from GPT-4o-mini with exact source citations. The Next.js chat frontend features message bubbles, source citation cards, confidence badges (green/amber/red), collection selector, and dark/light mode. Low-confidence answers are automatically logged as content gaps for documentation teams to address.
