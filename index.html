About

Documentation that thinks like engineering.

I'm a Documentation Engineer and Information Architect based in Bengaluru, India. My career spans 20 years across various domains including payment technology (Fundtech), asset management (SunGard), enterprise communications (Avaya), and financial crime prevention (NICE Actimize), where I led documentation for five enterprise platform solutions across on-prem and cloud deployments.

My documentation practice serves the full spectrum of enterprise software audiences. For engineers and developers, I produce REST API references built to OpenAPI specification, SDK integration guides with multi-language code samples, authentication and onboarding tutorials, webhook event catalogs, and error reference libraries — written not just to describe what an endpoint does, but why it exists, what can go wrong, and how to validate it end to end. For system administrators, I write deployment guides, configuration runbooks, and upgrade procedures. For front-end and business users, I produce task-based workflow documentation and UI reference material. For data and compliance analysts, I develop field reference guides and audit-ready documentation. For professional services consultants and client-side implementers, I build implementation playbooks and post-sale onboarding documentation. Delivery extends beyond written reference to include structured knowledge bases and narrated video walkthroughs that contextualise product functionality within real operational scenarios.

Beyond writing, I architect documentation ecosystems, embed docs into CI/CD pipelines, and shape developer portal roadmaps covering search, navigation, and content discoverability. I mentor teams on Git workflows, JIRA, and structured authoring. At NICE Actimize, I led the evaluation of seven AI and productivity tools, building governance controls that resulted in four tools being formally adopted across the team.

Outside my day job, I build documentation tooling: RepoDocs AI is an AI-prompt-powered docs-as-code framework for SaaS API teams using Claude; DocCraft AI converts raw inputs into polished documentation with GPT-4o; and my PPT-to-Video pipeline automates narrated documentation with Azure TTS and FFmpeg. I treat documentation as a product capability, not a deliverable that follows the release.
